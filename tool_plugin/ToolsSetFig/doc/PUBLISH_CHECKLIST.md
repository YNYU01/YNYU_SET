# Figma 社区发布检查清单 / Publishing Checklist

本文档用于确保插件发布前所有准备工作已完成。

## 📋 代码准备 / Code Preparation

### 构建检查
- [ ] 运行 `npx webpack` 成功构建
- [ ] `builds/code.js` 文件已生成
- [ ] `builds/ui.html` 文件已生成
- [ ] `builds/manifest.json` 文件已更新
- [ ] 所有文件无语法错误

### 功能测试
- [ ] 创建页：上传图片、表格、TSV、`.zy` 均能成功建帧
- [ ] 创建页：高级设置保存有效
- [ ] 导出页：三种导出类型都能生成记录
- [ ] 导出页：删除/重新上传与批量勾选无异常
- [ ] 表单页：预设表格 + 自定义组件均可生成 `@table`
- [ ] 表单页：映射文本、名称、属性、标签至少各测试一次
- [ ] 变量页：能读取当前文件样式/变量
- [ ] 变量页：示例与本地表双向同步成功
- [ ] 编辑页：可导入 & 应用至少一个滤镜栈
- [ ] 编辑页：导出 JSON 预设后再次导入依然有效
- [ ] 主题/语言/窗口尺寸：切换后记忆到下一次启动

### 兼容性测试
- [ ] Figma Desktop App 测试通过
- [ ] Figma Web 测试通过（如支持）
- [ ] 不同操作系统测试（Windows / macOS）
- [ ] 网络断开时的降级处理（如适用）

## 🎨 视觉资源准备 / Visual Assets

### 必需资源
- [ ] **插件图标**：128×128 像素，PNG 格式，透明背景
  - 文件名建议：`icon-128.png`
  - 设计要点：清晰、简洁、符合 Figma 风格

- [ ] **封面图**：1920×960 像素
  - 文件名建议：`cover-1920x960.png`
  - 安全区域：1600×960 像素（重要内容在此区域内）
  - 设计要点：展示插件核心功能，吸引用户

### 截图准备
- [ ] **主界面截图**：展示六个功能模块标签页
- [ ] **创建功能截图**：展示表格导入和批量创建过程
- [ ] **导出功能截图**：展示导出队列和多种格式选择
- [ ] **变量管理截图**：展示样式/变量同步界面
- [ ] **表格驱动截图**：展示数据映射和批量更新效果

### 视频准备（可选但推荐）
- [ ] **演示视频**：30-60 秒功能演示
  - 建议流程：导入表格→创建画板→批量修改→导出图片
  - 格式：MP4 或 GIF
  - 分辨率：至少 1280×720

## 🚀 发布流程 / Publishing Steps

### 发布前
- [ ] 所有检查项已完成
- [ ] 插件已在本地充分测试
- [ ] 文档已完善
- [ ] 视觉资源已准备

### 发布步骤
1. [ ] 在 Figma Desktop 中打开：`Plugins → Manage Plugins…`
2. [ ] 找到 "YN+ ToolsSet" 插件
3. [ ] 点击 "Publish New Release"
4. [ ] 填写版本号：`1.0.0`
5. [ ] 填写更新日志（从 CHANGELOG.md 复制）
6. [ ] 上传插件图标（128×128）
7. [ ] 上传封面图（1920×960）
8. [ ] 上传截图（至少 3-5 张）
9. [ ] 上传演示视频（可选）
10. [ ] 填写插件描述
11. [ ] 填写权限说明
12. [ ] 选择发布范围（整个 Figma 社区）
13. [ ] 确认所有信息无误
14. [ ] 点击发布

### 发布后
- [ ] 等待审核（通常 1-3 个工作日）
- [ ] 准备回复可能的审核反馈
- [ ] 监控用户反馈
- [ ] 准备后续版本更新
