# GitHub Issues 和 Discussions 快速开始

## 🚀 5分钟快速设置

### 第一步：启用功能

1. **进入你的 GitHub 仓库**
   - 访问：`https://github.com/YNYU01/YNYU_SET`

2. **启用 Issues**
   - 点击仓库顶部的 **Settings**（设置）
   - 在左侧菜单找到 **General**
   - 滚动到 **Features** 部分
   - ✅ 勾选 **Issues**
   - 点击 **Save changes**

3. **启用 Discussions**
   - 在同一个 **Features** 部分
   - ✅ 勾选 **Discussions**
   - 点击 **Save changes**

### 第二步：设置 Issue 模板（已自动创建）

模板文件已经创建在 `.github/ISSUE_TEMPLATE/` 目录下：
- ✅ `bug_report.md` - Bug报告模板
- ✅ `feature_request.md` - 功能请求模板
- ✅ `config.yml` - 配置文件

**这些文件已经提交到仓库，GitHub 会自动识别！**

### 第三步：创建标签（可选但推荐）

1. 进入 **Issues** 页面
2. 点击 **Labels** 链接
3. 点击 **New label** 创建以下标签：

**必需标签**：
- `bug` - 颜色：#d73a4a（红色）
- `enhancement` - 颜色：#a2eeef（绿色）
- `documentation` - 颜色：#0075ca（蓝色）
- `question` - 颜色：#d876e3（黄色）
- `good first issue` - 颜色：#7057ff（紫色）

### 第四步：设置 Discussions 类别

1. 进入 **Discussions** 页面
2. 点击 **New discussion** 旁边的 ⚙️ 图标
3. 点击 **Manage categories**
4. 创建以下类别：

- **💡 Ideas**（想法）- 黄色
- **❓ Q&A**（问答）- 蓝色
- **🎉 Show and Tell**（展示）- 绿色
- **📢 Announcements**（公告）- 红色（仅维护者）

### 第五步：测试

1. **测试 Issues**
   - 点击 **Issues** 标签
   - 点击 **New issue**
   - 你应该能看到模板选择界面

2. **测试 Discussions**
   - 点击 **Discussions** 标签
   - 点击 **New discussion**
   - 选择类别并创建第一个讨论

## ❓ 常见问题

### Q: 为什么我看不到 Issues/Discussions？
A: 确保在 Settings → Features 中已启用这些功能。

### Q: Issue 模板不显示？
A: 确保 `.github/ISSUE_TEMPLATE/` 目录下的文件已提交到仓库。

### Q: 如何删除不需要的标签？
A: 在 Labels 页面，点击标签旁边的垃圾桶图标。

### Q: 可以修改模板吗？
A: 可以！直接编辑 `.github/ISSUE_TEMPLATE/` 下的文件即可。
